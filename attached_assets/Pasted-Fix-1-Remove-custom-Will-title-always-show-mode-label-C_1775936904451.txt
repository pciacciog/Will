Fix 2 — Check-in calendar is unresponsive
In DailyCheckInModal.tsx, tapping a date on the calendar strip does nothing — the date does not get selected and the yes/partial/no options do not respond. This is a bug in the date selection handler.
Find the onDateSelect or equivalent handler inside DailyCheckInModal.tsx. Document exactly what it currently does and why it is not updating state when a date is tapped.
Common causes to check:

The date object being passed to the handler is a different format than what the component is comparing against — e.g. a Date object vs a "YYYY-MM-DD" string causing equality checks to always fail
The selected date state is being set but the component is not re-rendering because the reference is not changing
The tap target is too small or the touch event is being swallowed by a parent scroll container

Fix the handler so that:

Tapping a date visually selects it (highlighted with the blue border as shown)
The date label below updates to show the selected date ("Saturday, April 11")
The three options (Yes I did it / Partially done / No not today) become active and tappable
Selecting an option highlights it
Tapping "Save Check-in" submits POST /api/wills/:id/check-ins with { date: "YYYY-MM-DD", status: "yes" | "partial" | "no" }
On success the modal closes and progress stats refresh

Test this on a real device after fixing — confirm that selecting a date, picking a status, and saving all work end to end.
