Redesign the Team Will detail page structure across all Team Will formats 
("We Will" and "I Will") and all three Will types (Event, Recurring, Duration).

The core problem on all screens: member avatars appear in three separate 
sections creating redundancy. Consolidate to one unified MEMBERS section 
everywhere. Also address completion visibility and notifications.

---

UNIVERSAL PAGE STRUCTURE (apply to all Team Will screens):

Every Team Will detail page should follow this exact order:

1. COMMITMENT — see per-format specs below
2. MEMBERS — one single unified avatar section (see Change 1)
3. YOUR PROGRESS — check-in section specific to each Will type (see Change 2)
4. PROOF — unchanged, keep exactly as-is

Remove any avatar displays that exist outside the single MEMBERS section.
Remove "Team Today" section entirely across all formats and types.

---

FORMAT A: "WE WILL" (shared commitment)

COMMITMENT section:
- One shared commitment text card at the top labeled "OUR COMMITMENT"
- Unchanged from current design, just repositioned if needed

MEMBERS section (We Will):

Pre-start (Will not yet Active):
- Horizontal avatar strip: each member's avatar + name
- Below each name: "In ✓" (green) if fully committed, 
  "Pending" (gray) if invited but not yet committed

Active (Will has started):
- Remove "In ✓" / "Pending" labels — no longer relevant
- Avatar states reflect progress instead:
  - Completed / checked in: green ring + small checkmark badge
  - Not yet: neutral avatar, no ring
- Updates in real time for all members viewing the page

---

FORMAT B: "I WILL" (individual commitments)

COMMITMENT section:
- No shared commitment card — remove "OUR COMMITMENT" entirely
- Each member's individual commitment is shown inside the MEMBERS 
  section (see below)

MEMBERS section (I Will):
- Stack members vertically as rows or cards (not a horizontal strip) 
  so commitment text has room to breathe
- Each row contains: avatar + name + their individual commitment text 
  beneath their name (e.g. "I will run 3 miles every morning")
- Current user ("You") listed first
- Same lifecycle status logic as We Will:
  - Pre-start: "In ✓" or "Pending" below commitment text
  - Active: green ring + checkmark on completion, neutral otherwise

---

CHANGE 2: YOUR PROGRESS section — per type specs (both formats)

Event type (one-time completion):
- Keep "Did you complete this?" button as-is
- "Yes" → confirm flow → marks complete
- On completion: replace button with "You completed it ✓" in green
- Trigger green ring on current user's avatar in MEMBERS section
- Push notification to all other members: "🎉 [Name] completed it!"
- Show minimal completion log below MEMBERS once at least one member 
  has completed: "[Name] completed · Today at 7:32 AM" 
  (chronological, most recent first)

Recurring type (daily/weekly habit):
- Keep existing check-in mechanic (Did it / Did not) as-is
- Avatar ring state reflects today's check-in only, resets each period:
  - Checked in today: green ring
  - Not yet: neutral
- No completion log

Duration type (goal over a time range):
- Keep existing progress mechanic as-is
- Avatar ring reflects whether member has logged progress for 
  the current period

---

CHANGE 3: Completion notification (Event type only)

When any member confirms completion:
- Push notification to all other committed members:
  Title: "🎉 [Name] completed it!"
  Body: "[Will commitment text]" (We Will) or 
        "[Name]'s commitment text" (I Will)
  Tapping opens the Team Will detail page

---

Do not touch the PROOF section on any screen. Keep it exactly as-is.
