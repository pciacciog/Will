eplace the tracking screen with a new notifications screen — all Will setup flows
We are replacing the existing tracking screen entirely across all Will creation flows with a redesigned notifications screen. This affects StartWill.tsx (Solo and Public Will) and CreateTeamWill.tsx (Team Will — I Will and We Will). Do not touch any other screens or logic.

Context on what is being replaced
The current tracking screen asks the user to pick between three options: "Every Day," "Specific Days," and "Final Review Only." This maps to the checkInType field (daily, specific_days, final_review). This entire screen is being replaced. The new screen handles type selection AND notification setup in one unified step.

Schema changes — run these migrations first
In shared/schema.ts, add two new nullable fields to the wills table:
commitmentCategory  varchar(10)   null   default null
-- values: 'habit', 'abstain', 'mission'

milestones  text   null   default null
-- JSON array: [{ day: number, label: string }]
These are both nullable. No existing rows break. No existing queries break. Confirm the migration is just two ALTER TABLE ADD COLUMN statements before running.

The new screen — full specification
Create a new component: client/src/components/NotificationsSetup.tsx
This component receives the following props:
typescriptinterface NotificationsSetupProps {
  what: string        // the user's will statement — from the What step
  because: string     // the user's why — from the Why step
  onComplete: (data: NotificationsData) => void
  onBack: () => void
}

interface NotificationsData {
  commitmentCategory: 'habit' | 'abstain' | 'mission'
  reminderTime: string | null        // HH:MM — reminder before the moment (Habit + Abstain)
  checkInTime: string | null         // HH:MM — check-in after (Habit only)
  checkInType: 'daily' | 'specific_days' | 'final_review'  // derived automatically
  milestones: { day: number; label: string }[] | null       // Abstain only
  missionReminderTime: string | null  // HH:MM — optional daily nudge (Mission only)
  deadlineReminders: { threeDays: boolean; oneDay: boolean; dayOf: boolean }  // Mission only
}
The checkInType field is derived automatically based on commitmentCategory:

habit → checkInType: 'daily'
abstain → checkInType: 'final_review'
mission → checkInType: 'final_review'
